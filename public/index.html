The protocol, not the pep talk

slop-mop is built for repos where AI is already touching code. It does not ask for better vibes. It gives the agent a constrained operating rail and keeps the human review surface tight.

1

Swab the slice

Run fast gates after every meaningful edit so failure stays local and cheap.

2

Buff the PR

Turn CI state and review threads into a concrete resolution loop instead of random thrash.

3

Sail when unsure

Let the tool choose the right verb when the repo is mid-remediation and context is messy.
